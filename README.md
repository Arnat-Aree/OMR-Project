ระบบตรวจกระดาษคำตอบแบบปรนัย 100 ข้อ ผ่าน Web Application ที่พัฒนาขึ้นเพื่อแก้ปัญหาการตรวจข้อสอบที่ล่าช้าและเสี่ยงต่อความผิดพลาด 

**โครงงานนี้ไม่ได้ใช้ AI, Machine Learning หรือ Deep Learning แต่อย่างใด** แต่เป็นการประยุกต์ใช้องค์ความรู้ด้านการประมวลผลภาพขั้นพื้นฐาน (Classical Image Processing) ทางคณิตศาสตร์และเมทริกซ์มาใช้งานเพื่อแก้ปัญหาในโลกจริง (Real-world problems) ตามขอบเขตของรายวิชา

---

## ✨ ความสามารถหลัก (Features)

* **Pure Image Processing:** ประมวลผลรวดเร็วและใช้ทรัพยากรเครื่องน้อยมาก เพราะไม่ต้องโหลดโมเดล AI หนักๆ
* **Real-world Robustness:** รองรับภาพถ่ายกระดาษคำตอบจากกล้องมือถือทั่วไป ที่อาจมีปัญหาแสงเงาตกกระทบไม่เท่ากัน หรือมีจุดรบกวนบนหน้ากระดาษ
* **Web-based Dashboard:** มีหน้าเว็บ UI ที่ใช้งานง่าย สามารถอัปโหลดรูปภาพ กำหนดเฉลย และแสดงผลการตรวจทับลงบนรูปภาพ (Overlay) ให้ตรวจสอบความถูกต้องได้ทันที
* **Dockerized:** ติดตั้งและรันระบบได้ง่ายๆ ผ่าน Docker หมดปัญหาเรื่องเวอร์ชันของไลบรารีไม่ตรงกัน

## 🧠 การประยุกต์ใช้องค์ความรู้จากห้องเรียน

จุดเด่นของโครงงานนี้คือการนำทฤษฎี Image Processing มาแก้ปัญหาที่พบเจอจริงเวลาถ่ายรูปกระดาษคำตอบ โดยมีกระบวนการ (Pipeline) ดังนี้:

### 1. การปรับสมดุลแสง (Histogram Equalization - CLAHE)
* **ปัญหา:** ภาพถ่ายจากสมาร์ทโฟนมักมีเงาตกกระทบไม่เท่ากัน (เช่น เงาจากมือหรือโทรศัพท์บังกระดาษ)
* **การแก้ปัญหา:** ระบบใช้เทคนิค **Contrast Limited Adaptive Histogram Equalization (CLAHE)** เพื่อปรับความเปรียบต่างแบบแยกเฉพาะพื้นที่ (Local blocks ขนาด 8x8) ทำให้รอยดินสอเด่นชัดขึ้นและแยกออกจากพื้นหลังกระดาษได้ แม้ในจุดที่มีเงาดำพาดผ่าน

### 2. การลดสัญญาณรบกวน (Noise Reduction - Median Filter)
* **ปัญหา:** ภาพกระดาษที่สแกนหรือถ่ายมามักมีรอยเปื้อน ฝุ่น หรือจุดดำเล็กๆ (Salt and Pepper noise)
* **การแก้ปัญหา:** ระบบใช้ **Median Blur (k=5)** ซึ่งช่วยกำจัดจุดรบกวนเหล่านี้ได้ดีเยี่ยม โดยมีข้อดีกว่า Average Filter ตรงที่ "เส้นขอบของรอยดินสอ" จะไม่เบลอ ทำให้การคำนวณพื้นที่วงกลมในขั้นตอนสุดท้ายยังคงความแม่นยำ

### 3. การแยกวัตถุออกจากพื้นหลัง (Otsu's Thresholding)
* **ปัญหา:** การตั้งค่าเกณฑ์ความสว่าง (Threshold) แบบตายตัวใช้ไม่ได้ผลจริง เพราะรูปถ่ายแต่ละรูปมีความสว่างไม่เท่ากัน
* **การแก้ปัญหา:** ระบบใช้อัลกอริทึมของ **Otsu** เข้ามาวิเคราะห์ฮิสโตแกรมของภาพ และคำนวณหาค่า Threshold ที่เหมาะสมที่สุดโดยอัตโนมัติ เพื่อแยก "รอยดินสอ" ออกจาก "พื้นหลังกระดาษ" ได้อย่างสมบูรณ์แบบในทุกสภาพแสง

### 4. นิเวศวิทยาของภาพ (Morphological Closing)
* **ปัญหา:** บ่อยครั้งที่ผู้สอบฝนดินสอไม่เต็มวง หรือฝนเป็นเส้นๆ ทำให้เกิดรอยโหว่ตรงกลางในภาพไบนารี
* **การแก้ปัญหา:** ระบบใช้เทคนิค **Morphological Closing** (การทำ Dilation แล้วตามด้วย Erosion ทันที) โดยใช้โครงสร้างแบบวงรี (Elliptical Structuring Element) เพื่อถมรอยโหว่เหล่านั้นให้กลายเป็นกลุ่มก้อน (Blob) ที่ทึบสมบูรณ์ พร้อมสำหรับการนำไปคำนวณอัตราส่วนพิกเซล (Fill Ratio) เพื่อตัดสินว่าข้อนั้นถูกฝนหรือไม่

## 🚀 วิธีการติดตั้งและใช้งาน

โปรเจกต์นี้ถูกแพ็กเกจไว้ใน Docker ทำให้สามารถรันข้ามแพลตฟอร์มได้อย่างเสถียรโดยไม่ต้องติดตั้ง Dependencies ในเครื่องหลัก

### สิ่งที่ต้องเตรียม
* ติดตั้ง [Docker Desktop](https://www.docker.com/products/docker-desktop) ในเครื่องให้เรียบร้อย

### 1. การสร้างและรันเซิร์ฟเวอร์
เปิด Terminal เข้าไปที่โฟลเดอร์ของโปรเจกต์ แล้วใช้คำสั่งนี้:

```bash
  docker-compose up --build
```
(เคล็ดลับ: หากต้องการให้เซิร์ฟเวอร์ทำงานอยู่เบื้องหลัง ให้เติม -d ต่อท้ายคำสั่ง)

### 2. การเข้าใช้งานระบบ

เมื่อเซิร์ฟเวอร์รันสำเร็จ ให้เปิดเว็บเบราว์เซอร์แล้วเข้าไปที่:


```bash
  http://localhost:5000
```

### 3. ขั้นตอนการใช้งาน

* **อัปโหลดกระดาษคำตอบ:** อัปโหลดรูปภาพกระดาษคำตอบ (รองรับแบบฟอร์ม 100 ข้อ 4 คอลัมน์)

* **ตั้งค่าเฉลย:** พิมพ์เฉลยที่ถูกต้องลงในช่อง หรือคลิก "สุ่มเฉลยทดสอบ"

* **ตั้งค่า Parameter (ถ้าจำเป็น):** สามารถปรับเกณฑ์ Fill Threshold ได้ (ค่าเริ่มต้น 0.35 หมายถึง หากวงกลมมีความทึบสีดำเกิน 35% จะถือว่าตอบข้อนั้น)

* **ประมวลผล:** คลิกปุ่ม "ตรวจคำตอบผ่านเซิร์ฟเวอร์" ระบบประมวลผลภาพด้วย OpenCV ในฝั่ง Backend และส่งผลลัพธ์กลับมาแสดงผลทันที

### 4. การปิดระบบ

หากต้องการหยุดการทำงานของเซิร์ฟเวอร์ ให้ใช้คำสั่ง:

```bash
  docker-compose down
```
