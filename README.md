ระบบตรวจกระดาษคำตอบแบบปรนัย 100 ข้อ ผ่าน Web Application ที่พัฒนาขึ้นเพื่อแก้ปัญหาการตรวจข้อสอบที่ล่าช้าและเสี่ยงต่อความผิดพลาด 

**โครงงานนี้ไม่ได้ใช้ AI, Machine Learning หรือ Deep Learning แต่อย่างใด** แต่เป็นการประยุกต์ใช้องค์ความรู้ด้านการประมวลผลภาพขั้นพื้นฐาน (Classical Image Processing) ทางคณิตศาสตร์และเมทริกซ์มาใช้งานเพื่อแก้ปัญหาในโลกจริง (Real-world problems) ตามขอบเขตของรายวิชา

---

## ✨ ความสามารถหลัก (Features)

* **Pure Image Processing:** ประมวลผลรวดเร็วและใช้ทรัพยากรเครื่องน้อยมาก เพราะไม่ต้องโหลดโมเดล AI หนักๆ
* **Auto-Alignment (Homography):** มีระบบทาบและดึงภาพให้ตรงสัดส่วนอัตโนมัติ แก้ปัญหาการถ่ายภาพเอียงหรือระยะขอบกระดาษที่คลาดเคลื่อนจากการพิมพ์
* **Shadow Robustness:** ทนทานต่อสภาพแสงเงาที่บกพร่อง ไม่ว่าจะเป็นเงาดำพาดผ่านกระดาษ หรือเงาจากแขนผู้ถ่าย ก็ยังสามารถแยกแยะรอยดินสอได้
* **Web-based Dashboard:** มีหน้าเว็บ UI ที่ใช้งานง่าย สามารถอัปโหลดรูปภาพ กำหนดเฉลย และแสดงผลการตรวจทับลงบนรูปภาพ (Overlay) ให้ตรวจสอบความถูกต้องได้ทันที
* **Dockerized:** ติดตั้งและรันระบบได้ง่ายๆ ผ่าน Docker หมดปัญหาเรื่องเวอร์ชันของไลบรารีไม่ตรงกัน

## 🧠 การประยุกต์ใช้องค์ความรู้จากห้องเรียน

จุดเด่นของโครงงานนี้คือการนำทฤษฎี Image Processing มาแก้ปัญหาที่พบเจอจริงเวลาถ่ายรูปกระดาษคำตอบ โดยมีกระบวนการ (Pipeline) ดังนี้:

### 1. การจัดตำแหน่งภาพ (Feature Matching & Homography)
* **ปัญหา:** ภาพถ่ายจากสมาร์ทโฟนมักมีความเอียง บิดเบี้ยว (Perspective) และมีระยะขอบกระดาษที่ไม่เท่ากับไฟล์ต้นฉบับดิจิทัล
* **การแก้ปัญหา:** ระบบใช้อัลกอริทึม **ORB (Oriented FAST and rotated BRIEF)** ในการหาจุดเด่น (Features) บนภาพถ่าย แล้วนำไปจับคู่ (Match) กับภาพต้นแบบ (`answer sheet.png`) จากนั้นใช้การแปลง **Homography** เพื่อดึงยืดและทาบภาพถ่ายให้ตรงกับสัดส่วนมาตรฐานแบบพิกเซลต่อพิกเซล

### 2. การปรับสมดุลแสง (Histogram Equalization - CLAHE)
* **ปัญหา:** ภาพถ่ายจากสมาร์ทโฟนมักมีแสงสว่างไม่สม่ำเสมอ
* **การแก้ปัญหา:** ใช้เทคนิค **CLAHE (Contrast Limited Adaptive Histogram Equalization)** เพื่อปรับความเปรียบต่างแบบแยกเฉพาะพื้นที่ (Local blocks ขนาด 8x8) ทำให้รอยดินสอเด่นชัดขึ้นแม้ในจุดที่มืด

### 3. การลดสัญญาณรบกวน (Noise Reduction - Gaussian & Median Blur)
* **ปัญหา:** ภาพกระดาษมักมีรอยเปื้อน ฝุ่น หรือจุดดำเล็กๆ (Salt and Pepper noise)
* **การแก้ปัญหา:** ใช้ตัวกรองสัญญาณรบกวน เพื่อทำให้ภาพเนียนขึ้นและกำจัดจุดดำเล็กๆ ออกไป โดยที่เส้นขอบของวงกลมไม่สูญเสียรายละเอียด

### 4. การแยกวัตถุสู้กับเงาดำ (Adaptive Thresholding)
* **ปัญหา:** การตั้งค่า Threshold แบบ Global (เช่น Otsu) จะพังทันทีเมื่อเจอ "เงาแขน" ที่ดำทึบพาดผ่านกระดาษ เพราะระบบจะมองว่าเงานั้นคือรอยฝนดินสอขนาดใหญ่
* **การแก้ปัญหา:** ระบบใช้ **Adaptive Thresholding** ซึ่งจะเปรียบเทียบความสว่างของพิกเซลกับเพื่อนบ้านรอบๆ ตัวมันเอง (Local neighborhood) ทำให้ระบบสามารถเมินเฉยต่อเงาแขนที่พาดผ่านเป็นวงกว้าง และจับได้เฉพาะรอยดินสอที่มืดกว่ากระดาษที่อยู่ติดกันเท่านั้น

### 5. สัณฐานวิทยาของภาพ (Morphological Closing)
* **ปัญหา:** บ่อยครั้งที่ผู้สอบฝนดินสอไม่เต็มวง ทำให้เกิดรอยโหว่ตรงกลาง
* **การแก้ปัญหา:** ใช้เทคนิค **Morphological Closing** (Dilation ตามด้วย Erosion) ด้วยโครงสร้างแบบวงรี ถมรอยโหว่ให้กลายเป็นกลุ่มก้อน (Blob) ทึบสมบูรณ์ ก่อนนำไปคำนวณอัตราส่วนพื้นที่ (Fill Ratio) เพื่อตัดสินคะแนน

## 🚀 วิธีการติดตั้งและใช้งาน

โปรเจกต์นี้ถูกแพ็กเกจไว้ใน Docker ทำให้สามารถรันข้ามแพลตฟอร์มได้อย่างเสถียร

### สิ่งที่ต้องเตรียม
* ติดตั้ง [Docker Desktop](https://www.docker.com/products/docker-desktop) ในเครื่องให้เรียบร้อย

### 1. การสร้างและรันเซิร์ฟเวอร์
เปิด Terminal เข้าไปที่โฟลเดอร์ของโปรเจกต์ แล้วใช้คำสั่งนี้:

```bash
  docker-compose up --build
```
(เคล็ดลับ: หากต้องการให้เซิร์ฟเวอร์ทำงานอยู่เบื้องหลัง ให้เติม -d ต่อท้ายคำสั่ง)

### 2. การเข้าใช้งานระบบ

เมื่อเซิร์ฟเวอร์รันสำเร็จ ให้เปิดเว็บเบราว์เซอร์แล้วเข้าไปที่:


```bash
  http://localhost:5000
```

### 3. ขั้นตอนการใช้งาน

* **อัปโหลดกระดาษคำตอบ:** อัปโหลดรูปภาพกระดาษคำตอบ (รองรับแบบฟอร์ม 100 ข้อ 4 คอลัมน์)

* **ตั้งค่าเฉลย:** พิมพ์เฉลยที่ถูกต้องลงในช่อง หรือคลิก "สุ่มเฉลยทดสอบ"

* **ตั้งค่า Parameter (ถ้าจำเป็น):** สามารถปรับเกณฑ์ Fill Threshold ได้ (ค่าเริ่มต้น 0.35 หมายถึง หากวงกลมมีความทึบสีดำเกิน 35% จะถือว่าตอบข้อนั้น)

* **ประมวลผล:** คลิกปุ่ม "ตรวจคำตอบผ่านเซิร์ฟเวอร์" ระบบประมวลผลภาพด้วย OpenCV ในฝั่ง Backend และส่งผลลัพธ์กลับมาแสดงผลทันที

### 4. การปิดระบบ

หากต้องการหยุดการทำงานของเซิร์ฟเวอร์ ให้ใช้คำสั่ง:

```bash
  docker-compose down
```
